# 云函数部署指南

## 问题说明

当前小程序报错 `FunctionName parameter could not be found` 是因为云函数还没有部署到云端。

## 解决方案

### 方案一：部署云函数（推荐用于生产环境）

1. **打开微信开发者工具**
   - 确保已登录微信开发者账号

2. **创建云开发环境**
   - 点击工具栏的"云开发"按钮
   - 创建一个新的云开发环境（免费版即可）
   - 记录环境ID（例如：cloud1-xxx）

3. **配置环境ID**
   - 打开 `miniprogram/app.js` 文件
   - 修改 `env` 字段为你的环境ID：
     ```javascript
     this.globalData = {
       env: "你的环境ID",  // 修改这里
       userInfo: null,
       // ...
     };
     ```

4. **部署云函数**
   在微信开发者工具中，右键点击以下每个云函数文件夹，选择"上传并部署：云端安装依赖"：
   
   - `cloudfunctions/addUserMedal`
   - `cloudfunctions/createPost`
   - `cloudfunctions/createTeam`
   - `cloudfunctions/getQuestions`
   - `cloudfunctions/getRankings`
   - `cloudfunctions/getUserData`
   - `cloudfunctions/saveRecord`
   - `cloudfunctions/updateUserPoints`

5. **创建数据库集合**
   在云开发控制台的数据库中，创建以下集合：
   - `users` - 用户数据
   - `records` - 答题记录
   - `posts` - 社区帖子
   - `teams` - 战队信息
   - `notes` - 学习笔记

### 方案二：使用本地数据（推荐用于开发测试）

我已经为所有云函数调用添加了错误处理，即使云函数未部署，小程序也能正常运行：

- 用户数据会存储在本地缓存中
- 积分和勋章会在本地更新
- 所有功能都能正常使用

**注意**：使用本地数据时，数据不会同步到云端，更换设备或清除缓存后数据会丢失。

## 云函数说明

### getUserData
获取用户数据，包括：
- 用户等级
- 用户积分
- 获得的勋章
- 用户角色
- VIP状态
- 所属省份
- 所属战队

### updateUserPoints
更新用户积分和等级

### addUserMedal
为用户添加新勋章

### getRankings
获取排行榜数据

### getQuestions
获取题目数据

### saveRecord
保存答题记录

### createPost
创建社区帖子

### createTeam
创建战队

## 测试建议

1. 先使用方案二（本地数据）进行功能测试
2. 功能测试完成后，使用方案一部署云函数
3. 部署完成后，修改代码移除错误处理或保留作为容错机制

## 常见问题

**Q: 为什么会出现云函数调用失败？**
A: 云函数需要先部署到云端才能使用，未部署时调用会失败。

**Q: 本地数据和云端数据有什么区别？**
A: 本地数据只存储在当前设备，云端数据可以跨设备同步。

**Q: 如何查看云函数是否部署成功？**
A: 在云开发控制台的"云函数"页面可以看到已部署的云函数列表。

**Q: 云开发免费版有什么限制？**
A: 免费版有一定的调用次数和存储空间限制，但对于学习和小规模使用足够了。

# 微信云开发数据库集合配置指南

## 集合列表及权限配置

### 1. users（用户信息集合）
**权限类型**: 所有用户可读，仅创建者可写

**说明**: 存储用户的基本信息、积分、勋章等

**字段说明**:
- `_id` - 用户唯一标识
- `openid` - 微信openid
- `nickname` - 用户昵称
- `avatar` - 用户头像
- `rank` - 军衔（列兵/少尉/中尉/上尉/少校/中校/上校/少将/中将/上将/大将/元帅）
- `points` - 积分
- `medals` - 获得的勋章数组
- `role` - 角色（user/vip/admin/dataAdmin）
- `province` - 省份
- `teamId` - 所属战队ID
- `totalQuestions` - 总答题数
- `correctRate` - 正确率
- `studyDays` - 学习天数
- `createTime` - 创建时间
- `updateTime` - 更新时间

**索引**:
- `openid` - 唯一索引
- `points` - 普通索引
- `province` - 普通索引
- `teamId` - 普通索引

## 创建步骤

### 1. 打开云开发控制台
在微信开发者工具中，点击"云开发"按钮进入控制台

### 2. 进入数据库
点击左侧菜单的"数据库"

### 3. 创建集合
点击"添加集合"按钮，输入集合名称，选择权限类型

### 4. 按顺序创建以下集合

| 集合名称 | 权限类型 | 说明 |
|---------|---------|------|
| users | 所有用户可读，仅创建者可写 | 用户信息 |
| questions | 所有用户可读，仅管理员可写 | 题目数据 |
| posts | 所有用户可读，仅创建者和管理员可写 | 社区动态 |
| teams | 所有用户可读，仅创建者和管理员可写 | 战队信息 |
| notes | 所有用户可读，仅创建者和管理员可写 | 学习笔记 |
| battles | 所有用户可读，仅管理员可写 | 对战记录 |
| records | 仅创建者可读，仅创建者可写 | 答题记录 |
| rankings | 所有用户可读，仅管理员可写 | 排行榜 |

### 5. 创建索引（可选但推荐）
对于每个集合，根据上述索引说明创建相应的索引以提高查询性能

## 权限类型说明

### 所有用户可读，仅创建者可写
- 任何用户都可以读取数据
- 只有数据的创建者可以修改数据
- 适用于：用户个人信息

### 所有用户可读，仅管理员可写
- 任何用户都可以读取数据
- 只有管理员可以修改数据
- 适用于：题目、排行榜等公共数据

### 所有用户可读，仅创建者和管理员可写
- 任何用户都可以读取数据
- 数据的创建者和管理员可以修改数据
- 适用于：社区内容、战队信息

### 仅创建者可读，仅创建者可写
- 只有数据的创建者可以读取和修改数据
- 适用于：个人答题记录等隐私数据

## 注意事项

1. **索引创建**: 建议在集合创建后立即创建索引，以提高查询性能
2. **权限安全**: 根据实际业务需求调整权限设置，确保数据安全
3. **数据备份**: 定期备份重要数据，防止数据丢失
4. **性能优化**: 对于频繁查询的字段，务必创建索引
5. **数据验证**: 可以在云函数中添加数据验证逻辑，确保数据完整性

## 自定义权限规则（高级）

如果需要更细粒度的权限控制，可以使用自定义权限规则。例如：

```javascript
// teams 集合的自定义权限规则示例
{
  "read": true,
  "write": "doc.leaderId == auth.openid || doc.members.indexOf(auth.openid) > -1"
}
```

这样设置后，战队队长和成员都可以修改战队信息。
